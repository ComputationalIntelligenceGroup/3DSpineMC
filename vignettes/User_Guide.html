Description

Dendritic spines are small membranous protusions. They are receptors of excitatory synapses placed on the surface of some neuronal dendrites. Their morphology have captured the attention of neuroscientific community because it has been associated with brain funcionality. For example, it has been claimed that thin spines contribute to learning, while the biggest and steady spines are linked to the memory process. Disturbances of their morphology or density have been related to mental disorders such as schizophrenia, dementia or mental retardation. Thus, the clustering of dendritic spines is an interest approach to better understand the previously describe relation between morphology and functionality. The present R package provides a method to cluster and analize 3D dendritic spines according to their morphology.

Prerequirements

This software has been developed as an R package. Consequently, it is needed an R enviroment and internet connectivity to download additional package dependencies. R software can be downloaded from http://cran.rstudio.com/index.html. We suggested to install 64 bits version of R (if applicable) and RStudio (https://www.rstudio.com/products/rstudio/download/).

install.packages("path/to/file/SpineMC_0.1.tar.gz", repos = NULL, type="source")

Finally, to have accessed to the functionalities of 3DSpineMC you must load the package with the command:

library(SpineMC)

After that, 3DSpineMC is loaded in the R workspace so you can start to cluster dendritic spines and analyze the results. To test the package, a model and a dataset of morphological features (with some added white noise) are included as part of the package to try the functionalities.

Clustering

The common executing flow is based on obtaining a probabilistic clustering from the features computed on the surface of the spines with multiresolution Reeb graph. This dataset can be generated from the library 3DSpineMFE

model <- spineClustering(csvSpines = system.file("extdata", "data.csv", package = "SpineMC"), numClusters = c(2:10), scale = F)

This computation can take several days when the dataset has several thousands of rows. Also enable the scale flag can increase the computational time considerably.

Results

The previous computation returns as result a probabilistic model that contains the probability of belonging of each spine to each cluster. It is an mclust object (see (mclust)[https://cran.r-project.org/web/packages/mclust/vignettes/mclust.html] for more information). Some information about the groups of spines can be extracted from it.

For example it can be obtained an histogram of the number of spines by cluster running:

  clusters<-paste("Cluster",1:model$G)
  distribution<-as.numeric(table(model$classification))
   dfDistribution<-data.frame(values=distribution,clusters=clusters)
  dfDistribution$values<-as.numeric(as.character(distribution))

  ggplot(dfDistribution,aes(clusters,values,fill=as.factor(clusters))) +
    geom_bar(stat="identity")+xlab("Cluster distribution") + ylab("Num spines")  + guides(fill=guide_legend(title=NULL)) + theme(plot.title = element_text(lineheight = .8, face = "bold", size=20), axis.title.x = element_text(face="bold",size=16), axis.title.y =element_text(face="bold",size=16), axis.text =element_text(size=12), legend.text =element_text(size=12))

Also it can be checked the degree of membership of each spine to each cluster. Each row represents one spine and its cells are the probability of membership to the cluster in each column. The sum of all the values of a row is 1. To obtain the membership probability run:

MDS

To make visualization and interpretation of the groups of dendritic spines easier, distance between clusters in a 4D space can be scaled to a 2D space with multidimensional scaling. The next figure shows how close is the morphology of a cluster respect the morphology of the other clusters. Additionally, in those cases where the cluster is represented just as a point that means that all the spines in the cluster belongs to that cluster with a probability close to 1. However when there is a continuum of points between two clusters that means that there are some spines whose morphology is a mix of the two clusters and cannot be assigned to any of them certainly.

MDS<-computeMDS(model,2)
plotMDS(model,MDS)

Overlapping

Ideally, clustering should find well defined clusters. Thus, overlapping between cluster is undesirable. To measure the overlapping between cluster the next function can be applied:

computeOverlapping(model)

As result a table is generated where each value is the degree of overlapping between each pair of cluster where 1 is total overlapping and 0 is not overlap. As it can be expected, the diagonal of the table is 1 in all cases because it is measuring the overlapping of each cluster with itself.

BIC score is the heuristic score used by the algorithm to select the number of clusters. The higher is the score, the better is the cluster. To plot the BIC score obtained for each number of the clusters evaluated during the clustering process you can run.

#Plot the BIC score by the number of cluster and their model name.
plotBIC(model)

#Show the number of clusters that maximize BIC
print(paste("The number of clusters is",model$G))
## [1] "The number of clusters is 4"
